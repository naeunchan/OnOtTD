# 온옷티디 릴리즈 체크리스트

## 1. 로컬 검증

1. `npm run verify`
2. 생성된 `onottd.ait` 파일이 최신 빌드 결과인지 확인
3. `granite.config.ts`의 브랜드 아이콘 URL이 실제 배포용 값인지 확인

## 2. 샌드박스 QA

1. `npm run dev`
2. 샌드박스 앱에서 `intoss://onottd` 실행
3. Android는 `adb reverse tcp:8081 tcp:8081`
4. Android는 `adb reverse tcp:5173 tcp:5173`
5. 설정의 `샌드박스 날씨 모드`에서 아래 경로를 각각 저장 후 홈 확인
6. `실시간 날씨 강제`
7. `기본 지역 실황 강제`
8. `mock 데이터 강제`

## 3. 홈 화면 확인 포인트

1. 홈 상단의 날씨 출처 배지
2. 샌드박스 진단 카드의 설정 모드, 실행 모드, 위치 경로, 갱신 상태
3. 새로고침 버튼 문구가 현재 경로와 맞는지 확인
4. 미세먼지 `나쁨` 이상이면 마스크 반영
5. 자외선 `높음` 이상이면 선글라스와 양산 반영

## 4. 설정 화면 확인 포인트

1. 저장된 샌드박스 모드와 현재 선택값이 일치하는지 확인
2. `빌드 기본값으로 되돌리기` 동작 확인
3. `온보딩부터 다시 시작` 시 프로필과 샌드박스 모드가 모두 초기화되는지 확인

## 5. 배포 직전

1. 콘솔 업로드 전 최신 `main` 기준 브랜치인지 확인
2. `.env.example` 기준 환경변수 값 점검
3. 라이브 API URL이 `https`인지 확인
